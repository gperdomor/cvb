## Install Class Variance Builder

To add Class Variance Builder to your project just execute the following command:

```sh npm2yarn
npm i cvb
```

### Handling Style Conflicts

If you want to merge Tailwind CSS classes without conflicts, you may wish to [roll-your-own `cvb`](../api-reference#defineconfig) with the [`tailwind-merge`](https://github.com/dcastil/tailwind-merge) package:

<details>

<summary>Example with tailwind-merge</summary>

```ts
// cvb.config.ts
import { defineConfig } from 'cvb';
import { twMerge } from 'tailwind-merge';

export const { cvb, cx, compose } = defineConfig({
  hooks: {
    onComplete: (className) => twMerge(className),
  },
});
```

```ts
// components/button.ts
import { cx, cvb } from "../cvb.config";

export const button = cvb({
  // 1. `twMerge` strips out `bg-gray-200`…
  base: "font-semibold bg-gray-200 border rounded",
  variants: {
    intent: {
      // 2. …as variant `bg-*` values take precedence
      primary: "bg-blue-500 text-white border-transparent hover:bg-blue-600",
      secondary: "bg-white text-gray-800 border-gray-400 hover:bg-gray-100",
    },
  }
  defaultVariants: {
    intent: "primary",
  },
});

button();
// => "font-semibold border rounded bg-blue-500 text-white border-transparent hover:bg-blue-600 text-base py-2 px-4 uppercase"

cx("bg-gray-200", "bg-blue-500");
// => "bg-blue-500"
```

</details>
