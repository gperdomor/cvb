title: Slots
description: Slots are handy when you need to apply style variations to multiple parts of a component.
---

While using `cvb` might be enough for simple cases, `svb` are a better fit for more complex cases.

## Creating the Slot recipe

A slot recipe consists of these properties:

- `slots`: An array of component parts to style
- `base`: The base styles per slot
- `variants`: The different visual styles for each slot
- `defaultVariants`: The default variant for the component
- `compoundVariants`: The compound variant combination and style overrides for each slot.

<Callout type="warn">
  Credit: This API was inspired by multipart components in [Chakra
  UI](https://v2.chakra-ui.com/docs/styled-system/component-style#styling-multipart-components) and slot variants in
  [Tailwind Variants](https://cva.style/).
</Callout>

Letâ€™s transform our previous badge example into a slot recipe using `svb`.

```ts title="components/badge.recipe.ts"
import { svb } from 'cvb';

// [!code ++]
export const badge = svb({
  // [!code ++]
  slots: ['root', 'dot'], // [!code ++]
  // [!code ++]
  base: {
    // [!code ++]
    root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs', // [!code ++]
    dot: '', // [!code ++]
  }, // [!code ++]
  // [!code ++]
  variants: {
    // [!code ++]
    color: {
      // [!code ++]
      green: {
        root: 'bg-green-50 text-green-700', // [!code ++]
        dot: 'fill-green-500', // [!code ++]
      }, // [!code ++]
      // [!code ++]
      indigo: {
        root: 'bg-indigo-50 text-indigo-700', // [!code ++]
        dot: 'fill-indigo-500', // [!code ++]
      }, // [!code ++]
      // [!code ++]
      red: {
        root: 'bg-red-50 text-red-700', // [!code ++]
        dot: 'fill-red-500', // [!code ++]
      }, // [!code ++]
    }, // [!code ++]
    // [!code ++]
    size: {
      // [!code ++]
      sm: {
        root: 'px-1.5 py-0.5', // [!code ++]
        dot: 'size-1.5', // [!code ++]
      }, // [!code ++]
      // [!code ++]
      md: {
        root: 'px-2 py-1', // [!code ++]
        dot: 'size-2', // [!code ++]
      }, // [!code ++]
    }, // [!code ++]
  }, // [!code ++]
}); // [!code ++]

badge({ color: 'indigo', size: 'sm' });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50 text-indigo-700 px-1.5 py-0.5',
 *   dot: 'size-1.5 fill-indigo-500',
 * }
 */

badge({ color: 'red', size: 'md' });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-red-50 text-red-700 px-2 py-1',
 *   dot: 'size-2 fill-red-500',
 * }
 */
```

<Callout>Use of Tailwind CSS is optional</Callout>

### Boolean variants

You can also add boolean variants to a component. This is useful when you want to add a state variant e.g. `disabled`.

```ts title="components/badge.recipe.ts"
import { svb } from 'cvb';

export const badge = svb({
  slots: ['root', 'dot'],
  base: {
    root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs',
    dot: '',
  },
  variants: {
    color: {
      green: {
        root: 'bg-green-50 text-green-700',
        dot: 'fill-green-500',
      },
      indigo: {
        root: 'bg-indigo-50 text-indigo-700',
        dot: 'fill-indigo-500',
      },
      red: {
        root: 'bg-red-50 text-red-700',
        dot: 'fill-red-500',
      },
    },
    size: {
      sm: {
        root: 'px-1.5 py-0.5',
        dot: 'size-1.5',
      },
      md: {
        root: 'px-2 py-1',
        dot: 'size-2',
      },
    },
    // [!code ++]
    disabled: {
      true: { root: 'opacity-50' }, // [!code ++]
    }, // [!code ++]
    // [!code ++]
    flat: {
      true: { root: '' }, // [!code ++]
      false: { root: 'ring-1 ring-inset' }, // [!code ++]
    }, // [!code ++]
  },
});

badge({ color: 'indigo', size: 'sm', disabled: true });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50 text-indigo-700 px-1.5 py-0.5 opacity-50',
 *   dot: 'size-1.5 fill-indigo-500',
 * }
 */

badge({ color: 'red', size: 'md', flat: false });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-red-50 text-red-700 px-2 py-1 ring-1 ring-inset',
 *   dot: 'size-2 fill-red-500',
 * }
 */
```

### Compound slots variants

The `compoundVariants` property in slots recipes works similarly to the `compoundVariants` in the normal recipe and the
purpose is the same: apply style overrides to a slot based on the combination of variants.

Let's say you want to apply a different border to the badge root based on its `color` and the `flat` variant values,
here's how to do it:

```ts title="components/badge.recipe.ts"
import { svb } from 'cvb';

export const badge = svb({
  slots: ['root', 'dot'],
  base: {
    root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs',
    dot: '',
  },
  variants: {
    color: {
      green: {
        root: 'bg-green-50 text-green-700',
        dot: 'fill-green-500',
      },
      indigo: {
        root: 'bg-indigo-50 text-indigo-700',
        dot: 'fill-indigo-500',
      },
      red: {
        root: 'bg-red-50 text-red-700',
        dot: 'fill-red-500',
      },
    },
    size: {
      sm: {
        root: 'px-1.5 py-0.5',
        dot: 'size-1.5',
      },
      md: {
        root: 'px-2 py-1',
        dot: 'size-2',
      },
    },
    disabled: {
      true: { root: 'opacity-50' },
    },
    flat: {
      true: { root: '' },
      false: { root: 'ring-1 ring-inset' },
    },
  },
  // [!code ++]
  compoundVariants: [
    // apply when both green color and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: 'green', // [!code ++]
      flat: false, // [!code ++]
      class: { root: 'ring-green-600/20' }, // You can also use "className" // [!code ++]
    }, // [!code ++]
    // apply when both indigo color and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: 'indigo', // [!code ++]
      flat: false, // [!code ++]
      class: { root: 'ring-indigo-700/10' }, // You can also use "className" // [!code ++]
    }, // [!code ++]
    // apply when both red color and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: 'red', // [!code ++]
      flat: false, // [!code ++]
      class: { root: 'ring-red-600/10' }, // You can also use "className" // [!code ++]
    }, // [!code ++]
    // apply when both green or red color, and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: ['green', 'red'], // [!code ++]
      flat: false, // [!code ++]
      class: { root: 'font-medium' }, // You can also use "className" // [!code ++]
    }, // [!code ++]
  ], // [!code ++]
});

badge({ color: 'indigo', size: 'md', flat: false });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50 text-indigo-700 px-2 py-1 ring-1 ring-inset ring-indigo-700/10',
 *   dot: 'size-2 fill-indigo-500',
 * }
 */

badge({ color: 'green', size: 'md', flat: false });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-green-50 text-green-700 px-2 py-1 ring-1 ring-inset ring-green-600/20 font-medium',
 *   dot: 'size-2 fill-green-500',
 * }
 */
```

### Default Variants

The `defaultVariants` property is used to set the default variant values for the recipe. This is useful when you want to
apply a variant by default. Here's an example of how to use `defaultVariants`:

```ts title="components/badge.recipe.ts"
import { svb } from 'cvb';

export const badge = svb({
  slots: ['root', 'dot'],
  base: {
    root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs',
    dot: '',
  },
  variants: {
    color: {
      green: {
        root: 'bg-green-50 text-green-700',
        dot: 'fill-green-500',
      },
      indigo: {
        root: 'bg-indigo-50 text-indigo-700',
        dot: 'fill-indigo-500',
      },
      red: {
        root: 'bg-red-50 text-red-700',
        dot: 'fill-red-500',
      },
    },
    size: {
      sm: {
        root: 'px-1.5 py-0.5',
        dot: 'size-1.5',
      },
      md: {
        root: 'px-2 py-1',
        dot: 'size-2',
      },
    },
    disabled: {
      true: { root: 'opacity-50' },
    },
    flat: {
      true: { root: '' },
      false: { root: 'ring-1 ring-inset' },
    },
  },
  compoundVariants: [
    // apply when both green color and no flat are selected
    {
      color: 'green',
      flat: false,
      class: { root: 'ring-green-600/20' }, // You can also use "className"
    },
    // apply when both indigo color and no flat are selected
    {
      color: 'indigo',
      flat: false,
      class: { root: 'ring-indigo-700/10' }, // You can also use "className"
    },
    // apply when both red color and no flat are selected
    {
      color: 'red',
      flat: false,
      class: { root: 'ring-red-600/10' }, // You can also use "className"
    },
    // apply when both green or red color, and no flat are selected
    {
      color: ['green', 'red'],
      flat: false,
      class: { root: 'font-medium' }, // You can also use "className"
    },
  ],
  // [!code ++]
  defaultVariants: {
    color: 'indigo', // [!code ++]
    size: 'md', // [!code ++]
    flat: 'false', // [!code ++]
  }, // [!code ++]
});

badge();
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50 text-indigo-700 px-2 py-1 ring-1 ring-inset ring-indigo-700/10',
 *   dot: 'size-2 fill-indigo-500',
 * }
 */

badge({ color: 'green', flat: true });
/**
 * Result:
 * {
 *   root: 'inline-flex items-center rounded-md gap-x-1.5 text-xs bg-green-50 text-green-700 px-2 py-1',
 *   dot: 'size-2 fill-green-500',
 * }
 */
```

## Using the recipe

The returned value from the `svb` function is a function that can be used to apply the recipe to a component. Here's an
example of how to use the `badge` recipe:

```tsx title="components/badge.tsx"
import { badge } from './badge.recipe'; // [!code highlight]

const Button = () => {
  const styles = badge({ color: 'green', size: 'md' }); // [!code highlight]

  return (
    <span className={styles.root}>
      <svg className={styles.dot} viewBox="0 0 6 6" aria-hidden="true">
        <circle cx="3" cy="3" r="3" />
      </svg>
      Badge
    </span>
  );
};
```
