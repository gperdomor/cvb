Variants are defined using the `cvb` function which was inspired by [Class Variance Authority](https://cva.style/). The
`cvb` function which takes an object as its argument.

## Creating the Variant recipe

A recipe consists of four properties:

- `base`: The base styles for the component.
- `variants`: The different visual styles for the component
- `compoundVariants`: The different combinations of variants for the component
- `defaultVariants`: The default variant values for the component

<Callout type="warn">Credit: This API was inspired by [Class Variance Authority](https://cva.style/).</Callout>

To kick things off, letâ€™s build a **"basic"** badge recipe, using `cvb` to handle our CSS classes.
You can add variants by using the `variants` property. There's no limit to how many variants you can add.

```ts title="components/badge.recipe.ts"
import { cvb } from 'cvb';

// [!code ++]
export const badge = cvb({
  base: 'inline-flex items-center rounded-md gap-x-1.5 text-xs', // [!code ++]
  // [!code ++]
  variants: {
    // [!code ++]
    color: {
      green: 'bg-green-50 text-green-700', // [!code ++]
      indigo: 'bg-indigo-50 text-indigo-700', // [!code ++]
      red: 'bg-red-50 text-red-700', // [!code ++]
    }, // [!code ++]
    // [!code ++]
    size: {
      sm: 'px-1.5 py-0.5', // [!code ++]
      md: 'px-2 py-1', // [!code ++]
    }, // [!code ++]
  }, // [!code ++]
}); // [!code ++]

badge({ color: 'indigo', size: 'sm' });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50
 * text-indigo-700 px-1.5 py-0.5
 */

badge({ color: 'red', size: 'md' });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-red-50 text-red-700
 * px-2 py-1
 */
```

<Callout>Use of Tailwind CSS is optional</Callout>

### Boolean variants

You can also add boolean variants to a component. This is useful when you want to add a state variant e.g. `disabled`.

```ts title="components/badge.recipe.ts"
import { cvb } from 'cvb';

export const badge = cvb({
  base: 'inline-flex items-center rounded-md gap-x-1.5 text-xs',
  variants: {
    color: {
      green: 'bg-green-50 text-green-700',
      indigo: 'bg-indigo-50 text-indigo-700',
      red: 'bg-red-50 text-red-700',
    },
    size: {
      sm: 'px-1.5 py-0.5',
      md: 'px-2 py-1',
    },
    // [!code ++]
    disabled: {
      true: 'opacity-50', // [!code ++]
    }, // [!code ++]
    // [!code ++]
    flat: {
      true: '', // [!code ++]
      false: 'ring-1 ring-inset', // [!code ++]
    }, // [!code ++]
  },
});

badge({ color: 'indigo', size: 'sm', disabled: true });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50
 * text-indigo-700 px-1.5 py-0.5 opacity-50
 */

badge({ color: 'red', size: 'md', flat: false });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-red-50 text-red-700
 * px-2 py-1 ring-1 ring-inset
 */
```

### Compound variants

Compound variants are a way to combine multiple variants together to create more complex sets of styles. They are
defined using the `compoundVariants` property, which takes an array of objects as its argument. Each object in the array
represents a set of conditions that must be met in order for the corresponding styles to be applied.

For example, you might want to apply a border color to the badge based on its `color` and the `flat` variant values,
here's how to do it:

```ts title="components/badge.recipe.ts"
import { cvb } from 'cvb';

export const badge = cvb({
  base: 'inline-flex items-center rounded-md gap-x-1.5 text-xs',
  variants: {
    color: {
      green: 'bg-green-50 text-green-700',
      indigo: 'bg-indigo-50 text-indigo-700',
      red: 'bg-red-50 text-red-700',
    },
    size: {
      sm: 'px-1.5 py-0.5',
      md: 'px-2 py-1',
    },
    disabled: {
      true: 'opacity-50',
    },
    flat: {
      true: '',
      false: 'ring-1 ring-inset',
    },
  },
  // [!code ++]
  compoundVariants: [
    // apply when both green color and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: 'green', // [!code ++]
      flat: false, // [!code ++]
      class: 'ring-green-600/20', // You can also use "className" // [!code ++]
    }, // [!code ++]
    // apply when both indigo color and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: 'indigo', // [!code ++]
      flat: false, // [!code ++]
      class: 'ring-indigo-700/10', // You can also use "className" // [!code ++]
    }, // [!code ++]
    // apply when both red color and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: 'red', // [!code ++]
      flat: false, // [!code ++]
      class: 'ring-red-600/10', // You can also use "className" // [!code ++]
    }, // [!code ++]
    // apply when both green or red color, and no flat are selected // [!code ++]
    // [!code ++]
    {
      color: ['green', 'red'], // [!code ++]
      flat: false, // [!code ++]
      class: 'font-medium', // You can also use "className" // [!code ++]
    }, // [!code ++]
  ], // [!code ++]
});

badge({ color: 'indigo', size: 'md', flat: false });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50 text-indigo-700 px-2
 * py-1 ring-1 ring-inset ring-indigo-700/10
 */

badge({ color: 'green', size: 'md', flat: false });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-green-50 text-green-700 px-2 py-1
 * ring-1 ring-inset ring-green-600/20 font-medium
 */
```

Overall, using compound variants allows you to create more complex sets of styles that can be applied to your components
based on multiple conditions.

By combining simple variants together in this way, you can create a wide range of visual styles without cluttering up
your code with lots of conditional logic.

### Default Variants

The `defaultVariants` property is used to set the default variant values for the recipe. This is useful when you want to
apply a variant by default. Here's an example of how to use `defaultVariants`:

```ts title="components/badge.recipe.ts"
import { cvb } from 'cvb';

export const badge = cvb({
  base: 'inline-flex items-center rounded-md gap-x-1.5 text-xs',
  variants: {
    color: {
      green: 'bg-green-50 text-green-700',
      indigo: 'bg-indigo-50 text-indigo-700',
      red: 'bg-red-50 text-red-700',
    },
    size: {
      sm: 'px-1.5 py-0.5',
      md: 'px-2 py-1',
    },
    disabled: {
      true: 'opacity-50',
    },
    flat: {
      true: '',
      false: 'ring-1 ring-inset',
    },
  },
  compoundVariants: [
    // apply when both green color and no flat are selected
    {
      color: 'green',
      flat: false,
      class: 'ring-green-600/20',
    },
    // apply when both indigo color and no flat are selected
    {
      color: 'indigo',
      flat: false,
      class: 'ring-indigo-700/10',
    },
    // apply when both red color and no flat are selected
    {
      color: 'red',
      flat: false,
      class: 'ring-red-600/10',
    },
    // apply when both green or red color, and no flat are selected
    {
      color: ['green', 'red'],
      flat: false,
      class: 'font-medium',
    },
  ],
  // [!code ++]
  defaultVariants: {
    color: 'indigo', // [!code ++]
    size: 'md', // [!code ++]
    flat: 'false', // [!code ++]
  }, // [!code ++]
});

badge();
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-indigo-50 text-indigo-700 px-2
 * py-1 ring-1 ring-inset ring-indigo-700/10
 */

badge({ color: 'green', flat: true });
/**
 * Result:
 * inline-flex items-center rounded-md gap-x-1.5 text-xs bg-green-50 text-green-700 px-2 py-1
 */
```

## Using the recipe

The returned value from the `cvb` function is a function that can be used to apply the recipe to a component. Here's an
example of how to use the `badge` recipe:

```tsx title="components/badge.tsx"
import { badge } from './badge.recipe'; // [!code highlight]

const Button = () => {
  const styles = badge({ color: 'green', size: 'md' }); // [!code highlight]

  return <span className={styles}>Badge</span>;
};
```
